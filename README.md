# Mini-Job-Portal-System-Java-Console-Based

=>Java Console-Based Assignment – Mini Job Portal System

=>Objective: 
Develop a menu-driven console-based Job Portal application using Core Java, OOP concepts, and
Collections. This assignment must be implemented without using any frameworks or databases.

=>Concepts to Use- 
OOP (Inheritance, Polymorphism, Abstraction, Encapsulation)- Collections (List, Set, Map)- equals() & hashCode()- Exception Handling- Menu-driven console application
Package Structure (Mandatory)
com.jobportal
model (User, Admin, Company, Candidate, Job)
service (JobService)
main (JobPortalApp)

=>Functional Requirements
User Roles
1. Admin – View all jobs and view applicants per job
2. Company – Post jobs and close jobs
3. Candidate – View jobs and apply for jobs
   
=>Rules- 
- Job IDs must be unique
- A candidate must not apply for the same job more than once
- Candidate cannot apply for a CLOSED job
- Use Collections only (arrays not allowed)

=>Restrictions
- No Database
- No Frameworks (Spring, Hibernate, etc.)
- No Streams or Lambdas
- Console-based application onl
